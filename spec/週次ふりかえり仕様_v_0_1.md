# 週次ふりかえり仕様 v0.1

更新日: 2026-04-16

## 1. 概要

- Web アプリに専用画面 `/weekly-reflection` を追加する。
- 対象は `先週` のみとし、履歴一覧や過去週セレクタは持たない。
- 週の単位は `JST / ISO 週 / 月曜始まり` とする。

## 2. 集計対象

- v1 の集計対象は `quests`, `completions`, `skills` のみ。
- `quest.source === 'browsing'` のクエストと完了記録は週次ふりかえりの集計対象外。
- 閲覧、食事、Fitbit は v1 の本文集計に含めない。
- `activity_sessions`, `daily_activity_logs`, `weekly_activity_reviews` などの行動ログ基盤データは本仕様の集計対象外とする。
- 行動ログ側の週次振り返りは `spec/行動ログ基盤仕様_v_0_1.md` の `週次行動レビュー` を正本とする。

## 3. 画面構成

- ヘッダー: `YYYY-MM-DD 〜 YYYY-MM-DD`
- 総合サマリー: `クリア数`, `獲得 User XP`, `行動した日数`, `先週いちばん伸びたスキル`
- 7日サマリー: 月〜日の `件数` と `XP`
- デイリー継続状況: `先週 X日 / 先々週 Y日`
- 先週の主役クエスト: 上位 5 件、`先週 X回 / 先々週 Y回`
- 先週伸びたスキル: 上位 5 件
- Lily コメント
- 来週のおすすめ 1〜3 件
- 空状態: `先週はクエスト記録がありませんでした。来週は小さな 1 件から始めましょう。`

## 4. 生成タイミング

- `weekly_reflection` はアプリ初期化時には自動作成しない。
- `/weekly-reflection` を初めて開いた時に、その週キーのふりかえりが未生成なら生成する。
- 同じ `weekKey` はキャッシュを再利用し、再生成しない。

## 5. AI

- 週次ふりかえりの生成モデルは `OpenAI gpt-5.4` 固定。
- `activeProvider` は参照しない。
- OpenAI API キー未設定、オフライン、AI 無効時はテンプレート文にフォールバックする。
- 文体は `良かった点 -> 改善点は 1 つだけ -> 来週の提案` の順で、責めないトーンにする。

## 6. 保存仕様

- `AssistantMessage` に `periodKey?: string` を追加する。
- `weekly_reflection` のメッセージは `triggerType: 'weekly_reflection'` と `periodKey: 対象週キー` を持つ。
- `AppMeta` に `latestWeeklyReflection?: { weekKey, comment, recommendations, generatedAt, provider }` を追加する。
- `meta.lastWeeklyReflectionWeek` は `最後に閲覧済みの週次ふりかえりの週キー` を意味する。

## 7. 導線

- ホーム画面には `未読の先週ふりかえり` がある時だけ `先週のふりかえりを見る` CTA を表示する。
- 記録画面では `quests` タブかつ `week` フィルタ時に `先週のふりかえりを見る` 導線を表示する。
- 記録画面の導線は、先週データがある場合は常に表示し、未読時のみ `未読` バッジで強調する。

## 8. 音声

- 週次ふりかえり作成時に自動 TTS は再生しない。
- Lily コメントの音声再生は手動操作のみ。
