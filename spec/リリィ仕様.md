# リリィチャット機能 仕様 v0.1

更新日: 2026-03-23

---

## 0. 目的

- ユーザーがリリィと双方向で会話できるチャット機能を提供する
- リリィはユーザーのアクティビティログ（DB保存）を知識として活用し、成長状況を理解した上で会話する
- 通常のAIチャットと同様に、会話履歴の管理（新規作成・切り替え・削除）が可能

---

## 1. 前提

- リリィは既存の一方向メッセージ機能（クエスト完了時のコメント等）に加え、双方向チャットが可能になる
- チャットにはOpenAI Chat Completions APIを使用し、モデルは`gpt-5.4`固定
- OpenAI APIキーが設定済みであることが必須（未設定時は設定画面に誘導）
- 会話データはDynamoDB + localStorageで永続化（既存パターン踏襲）

---

## 2. 画面仕様

### 2-1. 導線

- ホーム画面のLilyセクション（最新コメントカード）に「リリィと話す」ボタンを配置
- ボタンタップで `/lily` に遷移
- BottomNavは変更しない

### 2-2. チャット画面レイアウト

```
┌──────────────────────────┐
│ ☰  　　　リリィ　　　  ＋ │  ← ヘッダー
├──────────────────────────┤
│                          │
│  [Lily] こんにちは！      │  ← アシスタントメッセージ（左寄せ）
│                          │
│      今日の調子は？ [User] │  ← ユーザーメッセージ（右寄せ）
│                          │
│  [Lily] 最近の記録を見る  │
│  と、運動スキルが伸びて   │
│  いますね！               │
│                          │
├──────────────────────────┤
│ [メッセージを入力...]  📤 │  ← 入力エリア（下部固定）
├──────────────────────────┤
│ ホーム クエスト ... 記録   │  ← BottomNav
└──────────────────────────┘
```

#### ヘッダー

| 要素 | 位置 | 動作 |
|------|------|------|
| ☰（ハンバーガーメニュー） | 左 | サイドドロワーを開く |
| 「リリィ」 | 中央 | タイトル表示 |
| ＋（新規チャット） | 右 | 新しい会話セッションを開始 |

#### メッセージエリア

- スクロール可能（`overflow-y-auto`）
- 新しいメッセージ送信時は自動で最下部にスクロール
- ユーザーメッセージ: 右寄せ、violet背景（`bg-violet-600 text-white`）、角丸
- アシスタントメッセージ: 左寄せ、Sparklesアイコン付き、白背景（`bg-white`）、角丸
- 送信中: タイピングインジケーター（ドットアニメーション）を表示

#### 入力エリア

- textarea（自動リサイズ、最大3行）
- 送信ボタン（空文字または送信中は無効化）
- Enterで送信、Shift+Enterで改行

### 2-3. ハンバーガーメニュー（サイドドロワー）

左からスライドインするドロワー:

```
┌─────────────────┐
│ 新規チャット  ＋  │
├─────────────────┤
│ 会話履歴         │
├─────────────────┤
│ 今日の振り返り    │  ← タイトル + 日付
│ 2026-03-22      │
│               🗑 │
├─────────────────┤
│ 運動について      │
│ 2026-03-21      │
│               🗑 │
└─────────────────┘
```

- 「新規チャット」: 新しいセッションを作成し、ドロワーを閉じる
- 会話履歴: `updatedAt`降順でセッション一覧を表示
- 各項目タップ: そのセッションに切り替え、ドロワーを閉じる
- 🗑（削除）: 確認なしでセッションと全メッセージを削除

### 2-4. OpenAI APIキー未設定時

入力エリアの代わりに以下を表示:

```
リリィとチャットするにはOpenAI APIキーの設定が必要です。
[設定画面を開く]
```

## 3. データモデル

### 3-1. ChatSession

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string | セッションID（`createId('chat')` で生成） |
| title | string | セッションタイトル（初回ユーザーメッセージの先頭30文字） |
| createdAt | string | ISO 8601 (JST) |
| updatedAt | string | ISO 8601 (JST) |

### 3-2. ChatMessage

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string | メッセージID（`createId('cmsg')` で生成） |
| sessionId | string | 所属セッションのID |
| role | `'user' \| 'assistant'` | 送信者 |
| content | string | メッセージ本文 |
| createdAt | string | ISO 8601 (JST) |

### 3-3. DynamoDB SK設計

- セッション: `PK=user#<userId>`, `SK=CHAT_SESSION#<sessionId>`
- メッセージ: `PK=user#<userId>`, `SK=CHAT_MSG#<sessionId>#<messageId>`
- メッセージの横断日付検索用に `GSI1PK=CHAT_MSG#<userId>`, `GSI1SK=<createdAt の epoch ms>` を保持する
- `GSI1SK` は ISO 文字列ではなく正規化済み epoch ms を使い、タイムゾーン表記ゆれの影響を避ける

## 4. API仕様

### 4-1. エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/chat-sessions` | 全セッション取得（updatedAt降順） |
| POST | `/chat-sessions` | セッション作成 |
| PUT | `/chat-sessions/{id}` | セッション更新（タイトル等） |
| DELETE | `/chat-sessions/{id}` | セッション＋全メッセージ削除 |
| GET | `/chat-sessions/{id}/messages` | セッションのメッセージ取得（時系列順） |
| GET | `/chat-messages?from=YYYY-MM-DD&to=YYYY-MM-DD` | JST 日付範囲の横断メッセージ取得 |
| POST | `/chat-sessions/{id}/messages` | メッセージ追加 |

### 4-2. 認証

既存のCognito JWT認証を使用（API Gatewayの`defaultAuthorizer`）。

## 5. AI連携仕様

### 5-1. 使用API

- OpenAI Chat Completions API (`POST https://api.openai.com/v1/chat/completions`)
- モデル: `gpt-5.4`
- 既存のResponses API（構造化出力用）とは別の関数で呼び出す

### 5-2. システムプロンプト構成

リリィのシステムプロンプトには以下を含める:

1. **ペルソナ**: 温かく励ます成長パートナー。日本語で会話。簡潔だが親しみのある口調
2. **ユーザー情報**: 現在のレベル、総XP、上位スキル（名前・レベル・カテゴリ）
3. **アクティビティログ**: 直近7日のログをカテゴリ別に集計して要約
4. **クエスト完了履歴**: 直近7日の完了記録（クエスト名・日時）
5. **応答ルール**:
   - 応答は簡潔に（100〜200文字程度）
   - ユーザーの成長を具体的に認める
   - 押し付けがましくならない程度の提案
   - ユーザーのプライバシーに配慮（ログにない情報を推測で語らない）

### 5-3. 会話コンテキスト

- セッション内の全メッセージを会話履歴として送信
- メッセージ数が多い場合は直近30メッセージに制限
- システムプロンプト + 会話履歴 + 新規メッセージの順で送信
- Lily Desktop 側のプロフィール・スキル・クエスト・completion・activity log 取得は 30 秒 TTL のスナップショットキャッシュを使う
- `complete_quest` / `create_quest` / `delete_quest` 成功時は会話コンテキストキャッシュを即時 invalidate する

### 5-4. エラーハンドリング

- APIエラー: リトライ（最大3回、指数バックオフ）
- オフライン: 「オフラインのためリリィと会話できません」メッセージ表示
- APIキー未設定: 入力エリアを無効化し、設定画面への誘導メッセージを表示

## 7. Tool Search（Function Calling）

リリィがチャット内でツールを呼び出し、ユーザーのデータを取得できる機能。
OpenAI Chat Completions API の `tools` パラメータ（Function Calling）を使用する。

- 参考ドキュメント: https://developers.openai.com/api/docs/guides/tools-tool-search

#### 実装済みツール（8ツール）

##### 参照系ツール（5ツール）

| ツール名 | 説明 | 主要パラメータ |
|----------|------|-----------|
| `get_browsing_times` | Web閲覧時間データ（カテゴリ別・サイト別） | `period`: `today` / `week` / `month` |
| `get_user_info` | プロフィール・設定・メタ情報 | `type`: `profile` / `settings` / `meta` |
| `get_quest_data` | クエスト一覧・完了記録（IDも出力） | `type`: `quests` / `completions`, `status?`, `questType?`, `category?`, `period?`, `questId?` |
| `get_skill_data` | スキル一覧・個人スキル辞書 | `type`: `skills` / `dictionary`, `status?`, `category?` |
| `get_messages_and_logs` | メッセージ・AI設定・ログ・チャット履歴 | `type`: `assistant_messages` / `ai_config` / `activity_logs` / `chat_sessions` / `chat_messages`, `triggerType?`, `period?`, `date?`, `fromDate?`, `toDate?`, `sessionId?` |

- `chat_messages` で `sessionId` を指定した場合のみ `/chat-sessions/{id}/messages` を使う
- `chat_sessions` / `chat_messages` で JST 日付指定をした場合は `/chat-messages` の range API を使う

##### 操作系ツール（3ツール）

| ツール名 | 説明 | 主要パラメータ |
|----------|------|-----------|
| `complete_quest` | クエストをクリア（完了）する。あいまいタイトルマッチング対応 | `query`（必須）: 検索クエリ（タイトルの一部やキーワード）, `note?` |
| `create_quest` | クエストを新規作成 | `title`（必須）, `description?`, `questType?`（デフォルト: `repeatable`）, `isDaily?`（`repeatable` のときのみ有効）, `xpReward?`（デフォルト: 10）, `category?` |
| `delete_quest` | クエストを削除またはアーカイブ | `questId?`, `title?`（部分一致検索）, `mode?`: `delete`（デフォルト）/ `archive` |

###### complete_quest の仕様

- `query` であいまい検索：完全一致 → 部分一致 → 単語マッチの順に優先
- チャット経由では、モデルが `query` を組み立てる際に漢字・ひらがな・カタカナの表記ゆれ、言い換え、近いニュアンス差を許容してよい
- チャット経由では、ユーザー発話そのままだけでなく、最も近いクエストを表す短い検索クエリへ言い換えて `complete_quest` を呼んでよい
- 例：クエスト「朝のトマトジュース」に対して「トマトジュース飲んだ」でマッチ
- アクティブなクエストのみが対象（`status === 'active'`）
- スコア0.2未満は不一致とみなし、アクティブクエスト一覧を返す
- Web版: `completeQuest` ストアアクション経由（availability チェック・スキル解決含む）
- Desktop版: API に `POST /completions` で完了記録を送信、one_time クエストは `PUT /quests/{id}` でステータスも更新
- Desktop版のスキル解決は 2 段構え。`fixedSkillId` / `defaultSkillId` で即時解決できない場合、初回は `skillResolutionStatus = pending` で completion を保存する
- Desktop版で OpenAI が利用可能かつ `privacyMode !== 'no_ai'` の場合、保存後に非同期で再判定し、`resolved` / `needs_confirmation` / `unclassified` として `resolvedSkillId` / `candidateSkillIds` / `resolutionReason` を `PUT /completions/{id}` へ反映する
- Desktop版で再判定が `resolved` になり、かつ `skillMappingMode !== 'ask_each_time'` の場合は `defaultSkillId` も `PUT /quests/{id}` で更新する
- Desktop版で OpenAI が使えない場合、または AI 呼び出しに失敗した場合は、ローカル fallback 判定で最終状態まで確定させ、`pending` のまま残さない

###### create_quest の仕様

- `isDaily` は `questType === 'repeatable'` かつ `true` のときだけ保存する
- `questType === 'one_time'` の場合は `isDaily` を保持しない
- `skillMappingMode` は常に `ai_auto` で作成
- `privacyMode` は常に `normal` で作成
- `source` は常に `manual` で作成
- 作成後、アプリストア（`upsertQuest`）経由でlocalStorage + クラウドに永続化

###### delete_quest の仕様

- `questId` または `title` のいずれかを指定（`title` は部分一致検索）
- `title` で複数マッチした場合はエラーを返し、`questId` での指定を促す
- `mode=delete`: 完全削除（完了履歴がある場合は削除不可、アーカイブを提案）
- `mode=archive`: アーカイブ（非表示にするが履歴は保持）
- アプリストア（`deleteQuest` / `archiveQuest`）経由で永続化

#### データソース

- `get_browsing_times`, `activity_logs` → バックエンドAPI経由（DynamoDB）
- その他 → フロントエンドのZustand store（`ToolContext`経由で`executeTool`に渡される）
- `ai_config` のAPIキーは `maskApiKey()` でマスクされる（セキュリティ）

#### フロー

1. ユーザーがメッセージを送信
2. `sendLilyChatMessage` に `tools` 定義を渡して Chat Completions API を呼ぶ
3. レスポンスの `finish_reason` が `tool_calls` の場合:
   a. `executeTool()` でツールを実行（バックエンド API 呼び出し等）
   b. ツール結果を `role: 'tool'` メッセージとして追加
   c. 再度 Chat Completions API を呼び、最終テキスト応答を取得
4. テキスト応答をアシスタントメッセージとしてUIに表示

- ツールコールは最大1ラウンド（無限ループ防止）
- ツール実行の中間メッセージはUIには表示しない

#### 関連ファイル

- `src/lib/chat-tools.ts` — ツール定義（`CHAT_TOOLS`）とツール実行（`executeTool`）
- `src/lib/ai.ts` — `sendLilyChatMessage`（`ChatCompletionResult` 型で tool_calls をハンドリング）
- `src/store/chat-store.ts` — `sendMessage` 内のツールコールループ

## 8. 制限事項

- チャット機能はOpenAI APIキーが設定されている場合のみ利用可能
- `activeProvider`の設定に関わらず、チャットは常にOpenAI（GPT-5.4）を使用
- オフライン時はチャット不可（読み取り専用で過去の会話は閲覧可能）
- 会話データはDynamoDBに永続保存（TTLなし）
