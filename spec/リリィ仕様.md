# リリィチャット機能 仕様 v0.1

更新日: 2026-03-22

---

## 0. 目的

- ユーザーがリリィと双方向で会話できるチャット機能を提供する
- リリィはユーザーのアクティビティログ（DB保存）を知識として活用し、成長状況を理解した上で会話する
- 通常のAIチャットと同様に、会話履歴の管理（新規作成・切り替え・削除）が可能

---

## 1. 前提

- リリィは既存の一方向メッセージ機能（クエスト完了時のコメント等）に加え、双方向チャットが可能になる
- チャットにはOpenAI Chat Completions APIを使用し、モデルは`gpt-5.4`固定
- OpenAI APIキーが設定済みであることが必須（未設定時は設定画面に誘導）
- 会話データはDynamoDB + localStorageで永続化（既存パターン踏襲）

---

## 2. 画面仕様

### 2-1. 導線

- ホーム画面のLilyセクション（最新コメントカード）に「リリィと話す」ボタンを配置
- ボタンタップで `/lily` に遷移
- BottomNavは変更しない

### 2-2. チャット画面レイアウト

```
┌──────────────────────────┐
│ ☰  　　　リリィ　　　  ＋ │  ← ヘッダー
├──────────────────────────┤
│                          │
│  [Lily] こんにちは！      │  ← アシスタントメッセージ（左寄せ）
│                          │
│      今日の調子は？ [User] │  ← ユーザーメッセージ（右寄せ）
│                          │
│  [Lily] 最近の記録を見る  │
│  と、運動スキルが伸びて   │
│  いますね！               │
│                          │
├──────────────────────────┤
│ [メッセージを入力...]  📤 │  ← 入力エリア（下部固定）
├──────────────────────────┤
│ ホーム クエスト ... 記録   │  ← BottomNav
└──────────────────────────┘
```

#### ヘッダー

| 要素 | 位置 | 動作 |
|------|------|------|
| ☰（ハンバーガーメニュー） | 左 | サイドドロワーを開く |
| 「リリィ」 | 中央 | タイトル表示 |
| ＋（新規チャット） | 右 | 新しい会話セッションを開始 |

#### メッセージエリア

- スクロール可能（`overflow-y-auto`）
- 新しいメッセージ送信時は自動で最下部にスクロール
- ユーザーメッセージ: 右寄せ、violet背景（`bg-violet-600 text-white`）、角丸
- アシスタントメッセージ: 左寄せ、Sparklesアイコン付き、白背景（`bg-white`）、角丸
- 送信中: タイピングインジケーター（ドットアニメーション）を表示

#### 入力エリア

- textarea（自動リサイズ、最大3行）
- 送信ボタン（空文字または送信中は無効化）
- Enterで送信、Shift+Enterで改行

### 2-3. ハンバーガーメニュー（サイドドロワー）

左からスライドインするドロワー:

```
┌─────────────────┐
│ 新規チャット  ＋  │
├─────────────────┤
│ 会話履歴         │
├─────────────────┤
│ 今日の振り返り    │  ← タイトル + 日付
│ 2026-03-22      │
│               🗑 │
├─────────────────┤
│ 運動について      │
│ 2026-03-21      │
│               🗑 │
└─────────────────┘
```

- 「新規チャット」: 新しいセッションを作成し、ドロワーを閉じる
- 会話履歴: `updatedAt`降順でセッション一覧を表示
- 各項目タップ: そのセッションに切り替え、ドロワーを閉じる
- 🗑（削除）: 確認なしでセッションと全メッセージを削除

### 2-4. OpenAI APIキー未設定時

入力エリアの代わりに以下を表示:

```
リリィとチャットするにはOpenAI APIキーの設定が必要です。
[設定画面を開く]
```

## 3. データモデル

### 3-1. ChatSession

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string | セッションID（`createId('chat')` で生成） |
| title | string | セッションタイトル（初回ユーザーメッセージの先頭30文字） |
| createdAt | string | ISO 8601 (JST) |
| updatedAt | string | ISO 8601 (JST) |

### 3-2. ChatMessage

| フィールド | 型 | 説明 |
|-----------|-----|------|
| id | string | メッセージID（`createId('cmsg')` で生成） |
| sessionId | string | 所属セッションのID |
| role | `'user' \| 'assistant'` | 送信者 |
| content | string | メッセージ本文 |
| createdAt | string | ISO 8601 (JST) |

### 3-3. DynamoDB SK設計

- セッション: `PK=user#<userId>`, `SK=CHAT_SESSION#<sessionId>`
- メッセージ: `PK=user#<userId>`, `SK=CHAT_MSG#<sessionId>#<messageId>`

## 4. API仕様

### 4-1. エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/chat-sessions` | 全セッション取得（updatedAt降順） |
| POST | `/chat-sessions` | セッション作成 |
| PUT | `/chat-sessions/{id}` | セッション更新（タイトル等） |
| DELETE | `/chat-sessions/{id}` | セッション＋全メッセージ削除 |
| GET | `/chat-sessions/{id}/messages` | セッションのメッセージ取得（時系列順） |
| POST | `/chat-sessions/{id}/messages` | メッセージ追加 |

### 4-2. 認証

既存のCognito JWT認証を使用（API Gatewayの`defaultAuthorizer`）。

## 5. AI連携仕様

### 5-1. 使用API

- OpenAI Chat Completions API (`POST https://api.openai.com/v1/chat/completions`)
- モデル: `gpt-5.4`
- 既存のResponses API（構造化出力用）とは別の関数で呼び出す

### 5-2. システムプロンプト構成

リリィのシステムプロンプトには以下を含める:

1. **ペルソナ**: 温かく励ます成長パートナー。日本語で会話。簡潔だが親しみのある口調
2. **ユーザー情報**: 現在のレベル、総XP、上位スキル（名前・レベル・カテゴリ）
3. **アクティビティログ**: 直近7日のログをカテゴリ別に集計して要約
4. **クエスト完了履歴**: 直近7日の完了記録（クエスト名・日時）
5. **応答ルール**:
   - 応答は簡潔に（100〜200文字程度）
   - ユーザーの成長を具体的に認める
   - 押し付けがましくならない程度の提案
   - ユーザーのプライバシーに配慮（ログにない情報を推測で語らない）

### 5-3. 会話コンテキスト

- セッション内の全メッセージを会話履歴として送信
- メッセージ数が多い場合は直近30メッセージに制限
- システムプロンプト + 会話履歴 + 新規メッセージの順で送信

### 5-4. エラーハンドリング

- APIエラー: リトライ（最大3回、指数バックオフ）
- オフライン: 「オフラインのためリリィと会話できません」メッセージ表示
- APIキー未設定: 入力エリアを無効化し、設定画面への誘導メッセージを表示

## 6. ローカルストレージ

### 6-1. キャッシュ戦略

| キー | 内容 |
|------|------|
| `app.chatSessions` | セッション一覧（JSON配列） |
| `app.chatMessages.<sessionId>` | セッションごとのメッセージ一覧 |

### 6-2. 同期方針

- 画面表示時はlocalStorageから即座に表示
- バックグラウンドでAPIから最新データを取得し、差分があれば更新
- メッセージ送信時は楽観的更新（UI即時反映→API保存）

---

## 7. 制限事項

- チャット機能はOpenAI APIキーが設定されている場合のみ利用可能
- `activeProvider`の設定に関わらず、チャットは常にOpenAI（GPT-5.4）を使用
- オフライン時はチャット不可（読み取り専用で過去の会話は閲覧可能）
- 会話データはDynamoDBに永続保存（TTLなし）
