# 自分育成ゲーム 画面遷移図 v0.1

更新日: 2026-04-18

注記:
- 実装は `HashRouter` を使っているため、ブラウザ上の実 URL は `#/...` になる。
- 本仕様では論理 route を `/...` 表記で記載する。

## 1. 情報設計の要点

- 下部ナビは `ホーム / クエスト / 成長 / 記録` の 4 導線に固定する。
- 中央の `＋` は画面 route ではなく、追加アクションシートの入口とする。
- `ステータス` と `スキル` は `成長` に統合する。
- `記録` は固定ハブとし、`成長記録 / 行動ログ / 生活ログ / 週次ふりかえり` の入口をまとめる。
- `records` 配下は route 主体で構成し、`/records` で前回の子画面を復元しない。

## 3. 全体遷移図

```text
[ホーム]
  ├─ 下部ナビ → [クエスト]
  ├─ 下部ナビ → [成長]
  ├─ 下部ナビ → [記録]
  ├─ ＋ → [追加アクションシート]
  ├─ 今日のクエスト > クリア → [クエストクリアモーダル]
  ├─ 未読の先週ふりかえり → [/records/review/weekly]
  ├─ 今日のクリア回数を見る → [/records/growth?range=today]
  ├─ 成長記録を見る → [/records]
  └─ Lily 導線 → [/lily]

[クエスト]
  ├─ 下部ナビ → [ホーム / 成長 / 記録]
  ├─ ＋ → [追加アクションシート]
  ├─ クエストカード > クリア → [クエストクリアモーダル]
  └─ クエストカード > 編集 → [/quests/new]

[成長]
  ├─ 下部ナビ → [ホーム / クエスト / 記録]
  ├─ ＋ → [追加アクションシート]
  ├─ スキルカード > 詳細 → [スキル詳細シート]
  ├─ 週次ふりかえりを見る → [/records/review/weekly]
  └─ Lily / 設定 → [/lily] / [/settings]

[記録ハブ]
  ├─ 下部ナビ → [ホーム / クエスト / 成長]
  ├─ ＋ → [追加アクションシート]
  ├─ 成長記録 → [/records/growth?range=today]
  ├─ 行動ログ → [/records/activity/today]
  ├─ 生活ログ → [/records/life/nutrition?date=today]
  └─ 週次ふりかえり → [/records/review/weekly]

[追加アクションシート]
  ├─ クエスト追加 → [/quests/new]
  ├─ 食事登録 → [/meal]
  ├─ 外側タップ / Esc → 元画面へ戻る
  └─ 選択後 → シートを閉じて遷移
```

## 4. 画面責務

### ホーム
- 今日やることの確認に限定する。
- 今日のクエスト、今日の要約、未読の週次ふりかえり、Lily の最新コメントを置く。
- 成長の詳細やスキル一覧の正式な置き場にはしない。

### 成長
- 旧 `status` と `skills` を統合した正式画面とする。
- 全体ステータス、最近の伸び、六系統、補助サマリー、スキル一覧、スキル詳細シートを集約する。

### 記録ハブ
- 記録の入口だけを置く。
- 前回見ていた記録画面の自動復帰は行わない。

### 成長記録
- クエスト完了履歴の正式な置き場とする。
- `today / week / all` は query `range` で表現する。
- `閲覧 / 栄養 / 健康` のローカルタブは持たない。

### 行動ログ
- route 主体の導線を維持する。
- `today / day / calendar / search / review-year / review-week` を個別 route で扱う。
- `session / event`、`month`、`year`、`weekKey`、`period` は query または path で表現する。

### 生活ログ
- `nutrition`、`health`、`browsing` を dedicated route に分ける。
- 日付は query `date=YYYY-MM-DD` で扱う。
- `browsing` の期間切替は query `period=day|week|month|all` で扱う。

### 週次ふりかえり
- 正式 route は `/records/review/weekly` とする。
- ホーム、成長、成長記録からのリンクはすべてここへ寄せる。

## 5. クエストクリア導線

## 6. route 一覧

### Canonical route
- `/`

- `/growth`

- `/records/activity/review/week?weekKey=YYYY-Www`

- `/records/life/nutrition?date=YYYY-MM-DD`
  - 栄養ログ画面
- `/records/life/health?date=YYYY-MM-DD`
  - 健康ログ画面
- `/records/life/browsing?period=day|week|month|all&date=YYYY-MM-DD`
  - 閲覧集計画面
- `/records/review/weekly`
  - 週次ふりかえり画面
- `/lily`
  - Lily 画面
- `/settings`
  - 設定画面
- `/clear/:completionId`
  - クリア演出画面
- `/meal`
  - 食事登録画面
- `/meal/analyze`
  - 食事解析画面
- `/meal/confirm`

### Redirect-only alias
- `/status`
  - `/growth` にリダイレクトする
- `/skills`
  - `/growth` にリダイレクトする
- `/weekly-reflection`
  - `/records/review/weekly` にリダイレクトする
- `/records/quests`
  - `/records/growth` にリダイレクトする
  - `range` または旧 `filter` query がある場合は引き継ぐ
- `/records/activity/browsing`
  - `/records/life/browsing` にリダイレクトする
  - `period` と `date` query がある場合は引き継ぐ

## 7. 記録系画面の正式遷移

```text
[任意画面]
  └─ 下部ナビ「記録」 → [/records]

[/records]
  ├─ 成長記録 → [/records/growth?range=today]
  ├─ 行動ログ → [/records/activity/today]
  ├─ 生活ログ → [/records/life/nutrition?date=YYYY-MM-DD]
  └─ 週次ふりかえり → [/records/review/weekly]

[/records/growth?range=today|week|all]
  ├─ range 切替 → 同一 route の query 更新
  ├─ 先週のふりかえりを見る → [/records/review/weekly]
  └─ タブ「行動ログ」 → [/records/activity/today]

[/records/activity/today?view=session|event]
  ├─ タブ「成長記録」 → [/records/growth?range=today]
  ├─ 日付移動 / カレンダー選択 → [/records/activity/day/:dateKey]
  ├─ カレンダー → [/records/activity/calendar?month=YYYY-MM]
  ├─ 検索 → [/records/activity/search]
  ├─ 週次レビュー → [/records/activity/review/year]
  └─ session/event 切替 → query `view` 更新

[/records/life/nutrition?date=YYYY-MM-DD]
  ├─ 健康 → [/records/life/health?date=同日]
  ├─ 閲覧 → [/records/life/browsing?period=day&date=同日]
  └─ 日付移動 → query `date` 更新

[/records/life/health?date=YYYY-MM-DD]
  ├─ 栄養 → [/records/life/nutrition?date=同日]
  ├─ 閲覧 → [/records/life/browsing?period=day&date=同日]
  └─ 日付移動 → query `date` 更新

[/records/life/browsing?period=day|week|month|all&date=YYYY-MM-DD]
  ├─ 栄養 → [/records/life/nutrition?date=同日]
  ├─ 健康 → [/records/life/health?date=同日]
  ├─ period 切替 → 同一 route の query `period` 更新
  └─ 日付移動 → query `date` 更新

[/records/review/weekly]
  └─ 記録へ → [/records/growth?range=week]
```

## 8. desktop からの deep link 導線

```text
[リリィデスクトップ 右クリック]
  └─ Web を開く
      ├─ ホーム → [/]
      ├─ クエスト → [/quests]
      ├─ 成長 → [/growth]
      ├─ 成長記録 → [/records/growth?range=today]
      ├─ 今日の行動ログ → [/records/activity/today]
      ├─ 行動ログカレンダー → [/records/activity/calendar]
      ├─ 行動ログ検索 → [/records/activity/search]
      ├─ 閲覧集計 → [/records/life/browsing?period=day&date=today]
      ├─ 週次行動レビュー → [/records/activity/review/year]
      ├─ 週次ふりかえり → [/records/review/weekly]
      ├─ Lily → [/lily]
      └─ 設定 → [/settings]
```

- deep link は通常導線と同じ canonical route を使う。
- 認証前アクセス時は、ログイン後に元 route へ戻す。

## 9. 実装メモ

- 下部ナビは `＋` を含めて 5 カラム表示だが、正式なトップレベル画面は 4 本である。
- `＋` アクションシートは route を持たないため、ブラウザ戻る履歴のノイズにしない。
- 記録系は local state タブより route を優先し、共有・戻る・deep link を安定させる。
- JST 基準の日付 query を使う画面では、`date`、`month`、`weekKey` の解釈も JST 前提とする。
