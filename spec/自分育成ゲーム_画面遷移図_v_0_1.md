```text
  └─ クリアボタン押下
      ↓
[クエストクリアモーダル]
  ├─ キャンセル → 遷移元へ戻る
  └─ クリアする
      ↓
      （クエスト完了処理）
      ↓
[クリア演出画面]
  ├─ 記録を見る → [記録画面]
  ├─ 次のクエスト → [ホーム画面]
  └─ 閉じる/背景タップ ※入れる場合 → [ホーム画面]
```

## 4. 推奨遷移ルール

### 4-1. 下部ナビ
下部ナビは以下の5画面に常時表示する。

- ホーム
- クエスト一覧
- クエスト作成
- スキル一覧
- 記録

ただし以下には表示しない。
- クエストクリアモーダル
- クリア演出画面

### 4-2. モーダル遷移
クエストクリアモーダルは単独画面ではなく、以下のどちらからでも開ける。

- ホーム画面
- クエスト一覧画面

閉じた場合は、必ず元の画面へ戻る。

### 4-3. クリア演出
クエストクリア後は直接ホームへ戻さず、必ず一度クリア演出画面を見せる。

理由:
- 完了体験を「消化」ではなく「成長」として見せるため
- XP加算、スキル進捗、リリィコメントをまとめて見せるため

## 6. 実装メモ

### 6-1. 画面スタック
- 常設画面はタブ遷移
- クエストクリアモーダルはオーバーレイ
- クリア演出はフルスクリーン遷移

### 6-2. 遷移元保持
クエストクリアモーダルは `sourceScreen` を保持する。

候補:
- `home`
- `quest_list`

これにより、キャンセル時に確実に元画面へ戻せる。

### 6-3. クリア演出後の遷移先
- デフォルト: ホーム画面
- ボタン選択時: 記録画面
- 将来的には「次のおすすめクエスト詳細」へ遷移も可能

## 8. 追記: クリア回数詳細導線

```text
[ホーム画面]
  ├─ 今日のクリアカードをタップ → [記録画面(filter=today)]
  └─ 記録を見る → [記録画面]

[記録画面]
  ├─ 今日カードをタップ → [記録画面(filter=today)]
  ├─ 今週カードをタップ → [記録画面(filter=week)]
  └─ すべてカードをタップ → [記録画面(filter=all)]
```

- 新しい独立画面は追加しない。
- `記録` 画面の初期表示は `today` 扱いとする。
- `今日 / 今週 / すべて` は記録画面内のフィルタ切替として扱う。
