## 4. 推奨遷移ルール

### 4-1. 下部ナビ
下部ナビは以下の5画面に常時表示する。

- ホーム
- クエスト一覧
- クエスト作成
- スキル一覧
- 記録

ただし以下には表示しない。
- クエストクリアモーダル
- クリア演出画面

### 4-2. モーダル遷移
クエストクリアモーダルは単独画面ではなく、以下のどちらからでも開ける。

- ホーム画面
- クエスト一覧画面

閉じた場合は、必ず元の画面へ戻る。

### 4-3. クリア演出
クエストクリア後は直接ホームへ戻さず、必ず一度クリア演出画面を見せる。

理由:
- 完了体験を「消化」ではなく「成長」として見せるため
- XP加算、スキル進捗、リリィコメントをまとめて見せるため

## 6. 実装メモ

### 6-1. 画面スタック
- 常設画面はタブ遷移
- クエストクリアモーダルはオーバーレイ
- クリア演出はフルスクリーン遷移

### 6-2. 遷移元保持
クエストクリアモーダルは `sourceScreen` を保持する。

候補:
- `home`
- `quest_list`

これにより、キャンセル時に確実に元画面へ戻せる。

### 6-3. クリア演出後の遷移先
- デフォルト: ホーム画面
- ボタン選択時: 記録画面
- 将来的には「次のおすすめクエスト詳細」へ遷移も可能

## 8. 追記: クリア回数詳細導線

```text
[ホーム画面]
  ├─ 今日のクリアカードをタップ → [記録画面(filter=today)]
  └─ 記録を見る → [記録画面]

[記録画面]
  ├─ 今日カードをタップ → [記録画面(filter=today)]
  ├─ 今週カードをタップ → [記録画面(filter=week)]
  └─ すべてカードをタップ → [記録画面(filter=all)]
```

- 新しい独立画面は追加しない。
- `記録` 画面の初期表示は `today` 扱いとする。
- `今日 / 今週 / すべて` は記録画面内のフィルタ切替として扱う。

## 9. route 一覧

- `/`
  - ホーム画面
- `/status`
  - ステータス画面
- `/quests`
  - クエスト一覧画面
- `/quests/new`
  - クエスト作成画面
- `/skills`
  - スキル一覧画面
- `/records`
  - 記録画面の入口
  - 直近に見ていた `records` 配下 route を復元し、初回は `/records/quests?range=today`
- `/records/quests?range=today|week|all`
  - 成長記録画面
- `/records/activity/today?view=session|event`
  - 今日の行動ログ画面
- `/records/activity/day/:dateKey?view=session|event`
  - 特定日の行動ログ画面
- `/records/activity/calendar?month=YYYY-MM`
  - 行動ログカレンダー画面
- `/records/activity/review/year?year=YYYY`
  - 週次行動レビュー一覧画面
- `/records/activity/search`
  - 行動ログ検索画面
- `/records/activity/review/week?weekKey=YYYY-Www`
  - 週次行動レビュー詳細画面
- `/weekly-reflection`
  - 週次ふりかえり画面
- `/lily`
  - リリィチャット画面
- `/settings`
  - 設定画面

## 10. 記録系画面の正式遷移

```text
[任意画面]
  └─ 下部ナビ「記録」 → [/records]
                         ├─ 復元成功 → [最後に見ていた records 配下画面]
                         └─ 初回 / 復元失敗 → [/records/quests?range=today]

[/records/quests?range=today|week|all]
  ├─ タブ「行動ログ」 → [/records/activity/today]
  ├─ フィルタ today/week/all 切替 → 同一 route の query 更新
  ├─ 先週のふりかえりを見る → [/weekly-reflection]
  └─ 下部ナビで他画面へ

[/records/activity/today?view=session|event]
  ├─ タブ「成長記録」 → [/records/quests?range=today]
  ├─ 日付移動 / カレンダー選択 → [/records/activity/day/:dateKey]
  ├─ カレンダー → [/records/activity/calendar?month=YYYY-MM]
  ├─ 検索 → [/records/activity/search]
  ├─ 週次レビュー → [/records/activity/review/year]
  └─ session/event 切替 → query `view` 更新

[/records/activity/day/:dateKey?view=session|event]
  ├─ 今日へ戻る → [/records/activity/today]
  ├─ カレンダー → [/records/activity/calendar?month=YYYY-MM]
  ├─ 検索 → [/records/activity/search]
  ├─ 週次レビュー → [/records/activity/review/year]
  └─ session/event 切替 → query `view` 更新

[/records/activity/calendar?month=YYYY-MM]
  ├─ 前月 → query `month` を前月へ更新
  ├─ 次月 → query `month` を次月へ更新
  ├─ 年月ピッカー変更 → query `month` を指定年月へ更新
  ├─ 日付セル選択 → [/records/activity/day/:dateKey]
  ├─ 今日へ戻る → [/records/activity/today]
  ├─ 検索 → [/records/activity/search]
  └─ 週次レビュー → [/records/activity/review/year]

[/records/activity/review/year?year=YYYY]
  ├─ 前年 → query `year` を前年へ更新
  ├─ 次年 → query `year` を次年へ更新
  ├─ 年ピッカー変更 → query `year` を指定年へ更新
  ├─ 週カード選択 → [/records/activity/review/week?weekKey=YYYY-Www]
  ├─ 今日の行動ログへ → [/records/activity/today]
  ├─ カレンダー → [/records/activity/calendar?month=YYYY-MM]
  └─ 検索 → [/records/activity/search]

[/records/activity/review/week?weekKey=YYYY-Www]
  ├─ 年一覧へ戻る → [/records/activity/review/year?year=YYYY]
  ├─ 今日の行動ログへ → [/records/activity/today]
  ├─ カレンダー → [/records/activity/calendar?month=YYYY-MM]
  └─ 検索 → [/records/activity/search]
```

## 11. desktop からの deep link 導線

```text
[リリィデスクトップ 右クリック]
  └─ Web を開く
      ├─ ホーム → [既定ブラウザで `/` を開く]
      ├─ 今日の成長記録 → [既定ブラウザで `/records/quests?range=today` を開く]
      ├─ 今日の行動ログ → [既定ブラウザで `/records/activity/today` を開く]
      ├─ 行動ログカレンダー → [既定ブラウザで `/records/activity/calendar` を開く]
      ├─ 行動ログ検索 → [既定ブラウザで `/records/activity/search` を開く]
      ├─ 週次行動レビュー → [既定ブラウザで `/records/activity/review/year` を開く]
      ├─ 週次ふりかえり → [既定ブラウザで `/weekly-reflection` を開く]
      ├─ リリィチャット → [既定ブラウザで `/lily` を開く]
      └─ 設定 → [既定ブラウザで `/settings` を開く]
```

- deep link の遷移先は、Web アプリ内の通常導線でも使われる正式 route と同一にする。
- 認証前アクセス時はログイン後に元 URL へ戻す。
