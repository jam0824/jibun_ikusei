# 自分ステータス画面仕様 v0.1

更新日: 2026-04-12

## 1. 概要

- Web アプリに専用画面 `/status` を追加する。
- 目指すのは、異世界転生系の「ステータス画面っぽさ」と、このアプリの「実際の行動ログに根ざした成長実感」を両立すること。
- ページ全体のトーンは既存の Web アプリに合わせ、ステータス画面だけが別作品の UI に見えないようにする。
- ホームは「今日やること」、スキルは「分野別の育ち方」、記録は「元データ確認」に寄せる。
- ステータス画面は、その間にある「今の自分を 1 枚で読む総合サマリー」として位置づける。
- v1 では新しい保存データや独立ポイント体系を増やさず、既存の `user`, `skills`, `completions`, `fitbit`, `nutrition`, `browsing` から派生表示する。

## 2. 目的

- 今の自分の状態を一覧で把握できるようにする
- どこが伸びているか、どこが手薄か、次に何をすると流れがよくなるかを直感的に読めるようにする
- 「ゲームっぽい楽しさ」は出しつつ、数字の根拠が追える状態を保つ

## 3. 非目的

- 性格診断や人格評価を行うこと
- 根拠のない AI 採点でユーザーをジャッジすること
- できていない点を強く責める画面にすること
- ホーム、スキル、記録の役割を丸ごと置き換えること

## 4. 画面の立ち位置

- ホーム:
  - 今日の進捗と次の行動に最短で入る画面
- スキル:
  - 個別スキルを一覧・詳細で見る画面
- 記録:
  - クエスト、閲覧、栄養、健康の元データを見る画面
- ステータス:
  - それらを横断して「今の自分」を読む画面

## 5. 導線

- ルートは `/status`
- 下タブは増やさず、導線はホームから入る
- ホームのレベルカード全体をタップ可能にし、`ステータスを見る` 導線を置く
- 将来的に必要ならホームのクイックアクションにも追加できるが、v1 では増やしすぎない

## 6. 画面構成

### 6-1. ステータスヘッダー

- 表示項目
  - 現在 Lv
  - 総 XP
  - 次レベルまでの残 XP
  - 連続達成日数
  - 現在の型
  - Lily の短いひとこと
- `現在の型` は、直近 30 日の系統 XP 上位 2 つから `学習 × 仕事型` のように表示する
- Lily コメントは専用生成を必須にせず、AI 未設定時はテンプレート文でよい

### 6-2. 六系統パラメータ

- 既存カテゴリを、ステータス画面では以下の表示名で見せる
  - 学習 -> 知識
  - 運動 -> 体力
  - 仕事 -> 実務
  - 生活 -> 生活
  - 対人 -> 対話
  - 創作 -> 創造
- 各パラメータカードに表示する項目
  - Lv
  - 累積 XP
  - 直近 7 日の増分
  - 代表スキル 1 件
- `その他` カテゴリは六系統には含めず、画面下部に `その他の成長` として別表示する

### 6-3. 今日のコンディション（補助情報）

- まずは「評価」より「把握」を優先する
- 六系統パラメータとは分離した補助情報として扱い、主要な成長指標には混ぜない
- 常時表示する項目
  - 今日のクリア回数
  - 今日の獲得 XP
  - 今週の行動日数
  - 直近の達成時刻
- 追加データがある場合のみ表示する項目
  - 健康: 睡眠時間、歩数、安静時心拍
  - 栄養: `不足 / 適正 / 過剰` の件数要約
  - 閲覧: 成長系閲覧時間、非成長系閲覧時間
- 未取得データは `未連携` `未登録` `データなし` と明示し、無理に良し悪しを判定しない

### 6-4. 最近の伸び

- `直近 7 日で伸びた上位 3 系統` を表示する
- 各項目には以下を表示する
  - 系統名
  - 増分 XP
  - 伸びの中心になったスキル名
- ここは「何が伸びているか」を見る領域であり、総量ではなく最近の変化を優先する

### 6-5. 次の一手

- ホームと同じおすすめクエストロジックを使い、最大 3 件を表示する
- それぞれに以下を表示する
  - クエスト名
  - 獲得 XP
  - 関連する系統
- `先週のふりかえり` が未読なら、ここにも小さく導線を出す

## 7. 派生ルール

### 7-1. 系統累積 XP

- 系統累積 XP は `status === active` のスキルだけを対象とする
- 計算式:
  - `categoryTotalXp = そのカテゴリに属する active skill の totalXp 合計`
- スキル統合済みデータは統合先に寄せて扱う

### 7-2. 系統 Lv

- 系統ごとの表示はランクではなく Lv を使う
- 理由:
  - 続けるほど最終的に全系統が高くなる設計では、`S` のような固定ランクより Lv のほうが自然
  - 既存のユーザー Lv / スキル Lv とも揃えやすい
- v1 の系統 Lv は既存スキル Lv と同じテンポに揃える
  - `categoryLevel = floor(categoryTotalXp / 50) + 1`
- 元になる累積 XP も必ず併記する

### 7-3. 現在の型

- `現在の型` は、直近 30 日で獲得した系統 XP の上位 2 系統で決める
- 計算式:
  - `categoryRecent30dXp = 直近 30 日の active completion 由来 skillXpAwarded をカテゴリ単位で合算した値`
- 例
  - `学習 × 仕事型`
  - `運動 × 生活型`
- 2 位が 0 XP の場合は 1 系統のみで表示する
  - 例: `学習型`
- 直近 30 日が全系統 0 XP の場合は型を固定表示せず、`最近の記録が増えると表示されます` のプレースホルダを出す
- 同点時は `系統累積 XP`, それも同じなら固定順 `学習 -> 運動 -> 仕事 -> 生活 -> 対人 -> 創作` で決める

### 7-4. 直近 7 日増分

- `直近 7 日増分` は `completedAt` が直近 7 日以内の active completion を対象とする
- `skillXpAwarded` があるものだけを集計する
- 集計時は `resolvedSkillId` を統合先スキルに解決してからカテゴリへ合算する

### 7-5. 今週の行動日数

- `今週の行動日数` は JST / 月曜始まりの週で、1 件以上 active completion がある日数とする
- 閲覧由来クエストを含めるかどうかは v1 では `含めない`
- 理由:
  - 週次ふりかえり仕様と揃えやすい
  - 自発的に積んだ行動の見え方を優先したい

## 8. 空状態と部分状態

- まだクエスト完了が 1 件もない場合
  - `あなたのステータスはまだ記録されていません。最初の 1 件をクリアすると成長がここに現れます。`
- スキルが未生成の場合
  - 六系統カードはプレースホルダ表示にし、`クエストをクリアすると系統が育ち始めます` を出す
- 健康 / 栄養 / 閲覧の追加データがない場合
  - セクション自体は残してもよいが、`未連携` `未登録` を明示する
  - エラー見えにはしない

## 9. UI トーン

- 既存の Web アプリのトーンを踏襲し、配色・余白・カード設計はホームやスキル画面の延長として扱う
- 「ゲーム風」にはするが、過剰に中二病・重厚ファンタジーへ寄せすぎない
- かっこよさは以下で出す
  - 区切り線やラベルの整理
  - Lv 表示
  - ステータスカードの密度
  - 小さな称号感
- 逆に、根拠のない `STR 999` のような表現は避ける
- すべての数字は既存データへ戻れる前提にする

## 10. v1 実装方針

- v1 ではフロントエンドの派生表示で完結させる
- 必須の新規 API は追加しない
- 既存ロジックを再利用する
  - ユーザーレベル
  - 直近 7 日増分
  - 連続達成
  - おすすめクエスト
  - 週次ふりかえり未読判定
- 先に UI 骨組みを入れ、その後に健康 / 栄養 / 閲覧の補助表示を段階追加する

## 11. 将来拡張

- 称号を `学習 × 仕事型` から、もう少し自然な二つ名へ進化させる
- `最近落ちている系統` ではなく `整えたい系統` のようなやさしい提案表示
- ステータス画面からスキル詳細や記録へ深掘り導線を追加する
- 先月比 / 先週比 などの比較表示を追加する
