# 食事スクリーンショット解析機能 仕様書（確定版）

---

## 1. 概要

本機能はリリィWebアプリ（自分育成アプリ）において、スマートフォンのスクリーンショットから栄養素情報を抽出し、データベースに登録する機能である。

---

## 2. 画面遷移

### フッター
- 「追加」ボタンを押下
  - クエスト追加タブ
  - 食事登録タブ
- デフォルトはクエスト追加タブ

---

## 3. 食事登録画面

### 基本仕様
- 登録日（デフォルト：当日）
- タップでカレンダー表示・日付変更可能

### 登録区分
- 1日分（daily）
- 朝（breakfast）
- 昼（lunch）
- 夜（dinner）

各項目をタップすると：
1. 画像選択
2. 解析実行
3. 抽出結果確認
4. 保存

---

## 4. スクリーンショット解析

### 対象画像
- スマートフォンのスクリーンショットのみ
- 以下条件を満たさない場合はエラー
  - 「1日分」の表示がある
  - 16栄養素の一覧が確認できる

### 使用モデル
- GPT-5（クライアントからAPI実行）

### 抽出対象

| 栄養素 | 単位 |
|--------|------|
| エネルギー | kcal |
| たんぱく質 | g |
| 脂質 | g |
| 糖質 | g |
| カリウム | mg |
| カルシウム | mg |
| 鉄 | mg |
| ビタミンA | ug |
| ビタミンE | mg |
| ビタミンB1 | mg |
| ビタミンB2 | mg |
| ビタミンB6 | mg |
| ビタミンC | mg |
| 食物繊維 | g |
| 飽和脂肪酸 | g |
| 塩分 | g |

### 抽出項目
- 摂取量（number）
- ラベル（不足 / 適正 / 過剰）
- 基準値（正規化）
  - 下限値
  - 上限値
  - 判定種別（range / min_only / max_only）

### 失敗時
- エラーメッセージ表示
- 保存不可

---

## 5. 確認画面

### 仕様
- 全栄養素を表示
- 以下のみ編集可能
  - 摂取量

### 未取得
- 未取得項目は「未取得」と表示
- 未取得のまま保存可能

---

## 6. 保存仕様

### DB
- DynamoDB

### 主キー
- ユーザーID + 日付 + 区分

### 区分値
- daily / breakfast / lunch / dinner

### 保存データ
- 摂取量（number）
- 単位（固定）
- ラベル（画像から取得そのまま）
- 基準値
  - 下限値
  - 上限値
  - 判定種別

### その他
- createdAt
- updatedAt

### 上書き
- 同一ユーザー + 日付 + 区分で上書き
- 上書き時は確認ダイアログ表示

### 画像
- 保存しない

---

## 7. データ表示（食事記録画面）

### 表示ルール
- 1日分が存在する場合
  → それを優先表示

- 1日分が存在しない場合
  → 朝/昼/夜を合算して表示

### 合算ルール
- 摂取量：単純合算
- 基準値：最初の1件を採用
- ラベル：再判定

### ラベル判定
- range：下限〜上限で判定
- min_only：下限以上で適正
- max_only：上限未満で適正

### 未取得
- 「未取得」と表示

### グラフ
- 横棒グラフ（栄養素ごと縦並び）

色分け
- 不足：青
- 適正：緑
- 過剰：赤

---

## 8. tool search

### 取得内容
- 指定日のデータ
- 以下をすべて含む
  - daily
  - breakfast
  - lunch
  - dinner

### 未取得
- "未取得" 文字列で返却

---

## 9. データ仕様

### 数値
- number型で保存

### 単位
- 固定値で保存
- ビタミンAは ug

### ラベル
- 画像の値をそのまま保存

---

## 10. その他仕様

- 同一画像でも再登録時は上書き
- 精度よりも実用性優先
- 数値の信頼度チェックは行わない
