# リリィシステム 機能アイディア

更新日: 2026-04-25

## 位置づけ

この文書は、既存のリリィシステム仕様を踏まえた将来機能のアイディアメモである。

現時点では実装確定仕様ではなく、今後の仕様化・優先順位づけ・プロトタイプ検討のための素材として扱う。

## 基本方針

- リリィは単なる AI チャットではなく、クエスト、行動ログ、健康、食事、閲覧、デスクトップ常駐を横断してユーザーを支える存在として育てる。
- 追加機能は、ユーザーの入力負荷を増やすよりも、既に集まっている記録を使って次の行動を軽くする方向を優先する。
- 提案や通知は、管理・監視ではなく「見守り」「気づき」「小さな再開」を主目的にする。
- 健康、行動、会話ログを扱う機能では、ユーザーが止められる、隠せる、消せることを前提にする。

## アイディア一覧

### 1. 今日の作戦会議

朝または PC 起動時に、睡眠、昨日の行動ログ、未完了クエスト、予定をもとに、リリィが今日の進め方を短く提案する。

- 例: `今日はこの3つだけで十分そうです`
- 入力候補:
  - 前日の `DailyActivityLog`
  - 今日のアクティブクエスト
  - Fitbit の睡眠・活動データ
  - 食事・体重データ
  - 直近の行動ログ傾向
- 期待効果:
  - ホーム画面を開いた瞬間に、今日やることを迷いにくくする
  - コンディションに合わない無理な提案を避ける

### 2. リリィからの自然なリマインド

固定時刻の通知ではなく、記録や流れに応じて控えめに声をかける。

- 例:
  - `昨日はこの時間に学習が進んでいました`
  - `今日はまだ体重計測クエストが残っています`
  - `50分くらい集中が続いているので、一度区切ってもよさそうです`
- 既存仕様との接続:
  - Desktop の自動雑談
  - Chrome 拡張の閲覧警告
  - クエスト完了履歴
  - 行動ログの ActivitySession
- 注意点:
  - 通知頻度を必ず制御する
  - 通常会話中や可聴タブ視聴中の割り込みは避ける

### 3. クエスト自動棚卸し

長く触っていないクエスト、達成率が高すぎるクエスト、実態とズレたクエストをリリィが見つけて整理を提案する。

- 例:
  - `このクエスト、最近は少し重そうです。5分版にしますか？`
  - `これはもう習慣化していそうなので、報酬を少し下げてもよさそうです`
  - `しばらく使っていないのでアーカイブ候補にしますか？`
- 実装候補:
  - クエストごとの直近完了日
  - 30日完了回数
  - 連続未達日数
  - 完了時メモや行動ログとの一致度
- 期待効果:
  - クエスト一覧が古い TODO で重くならない
  - ユーザーの実態に合う軽い運用を保てる

### 4. 習慣の兆し検出

ユーザー自身がまだ気づいていない行動パターンを、リリィがやわらかく言語化する。

- 例:
  - `最近、夜に創作の時間が増えています`
  - `週末は健康系の行動が入りやすいみたいです`
  - `午前中は調査、午後は実装に寄りやすい流れがあります`
- 入力候補:
  - ActivitySession のカテゴリ・時間帯
  - WeeklyActivityReview
  - クエスト完了曜日・時間帯
  - Fitbit / 食事 / 体重の変化
- 注意点:
  - 評価や診断のように見せない
  - `こういう傾向かもしれません` 程度の表現に留める

### 5. コンディション連動クエスト

睡眠、歩数、心拍、食事、体重などの状態をもとに、その日のおすすめクエストを変える。

- 例:
  - 睡眠が短い日は高負荷クエストより回復・整理系を優先する
  - 活動量が少ない日は軽い散歩やストレッチを提案する
  - 栄養で不足が目立つ日は食事登録や買い物メモを提案する
- 既存仕様との接続:
  - ステータス画面の今日のコンディション
  - Fitbit データ
  - 食事スクリーンショット解析データ
  - Health Planet データ
- 注意点:
  - 医療的な助言にしない
  - 断定ではなく、行動候補として提示する

### 6. 未完了の流れを拾う機能

行動ログから、調べかけ、書きかけ、途中で止まった作業を抽出し、後日リリィが再開のきっかけを作る。

- 例:
  - `昨日のReact調査、続きを10分だけやりますか？`
  - `途中で止まっていたメモがありそうです`
  - `この作業、クエストにしておくと拾いやすそうです`
- 入力候補:
  - ActivitySession の短時間中断
  - 同じ projectName / domain / appName の再出現
  - DailyActivityLog の mainThemes
  - 手動メモ
- 期待効果:
  - 作業の再開コストを下げる
  - 行動ログが単なる記録ではなく、次の行動の入口になる

### 7. リリィの記憶帳

会話、行動ログ、クエストから、ユーザーの好みや大事にしているテーマを長期メモとして育てる。

- 例:
  - 興味分野
  - よく取り組む時間帯
  - 苦手な通知タイミング
  - 好きな励まされ方
  - 続きやすいクエスト粒度
- 用途:
  - 雑談の種選び
  - クエスト提案
  - 週次・月次ふりかえり
  - リマインド文面の調整
- 必須要件:
  - ユーザーが記憶内容を閲覧できる
  - 編集・削除できる
  - AI に使わない設定を選べる

### 8. 月次ふりかえり / 成長年表

週次より大きな粒度で、今月伸びたスキル、よく出たテーマ、生活の変化をまとめる。

- 例:
  - `今月の主役スキル`
  - `増えた行動 / 減った行動`
  - `よく出たテーマ`
  - `コンディションの流れ`
  - `来月に残したい問い`
- 表示案:
  - 月次サマリー
  - 成長年表
  - 主要クエストのハイライト
  - Lily の観察メモ
- 既存仕様との接続:
  - WeeklyActivityReview
  - 週次ふりかえり
  - ステータス画面
  - スキル一覧

### 9. クエストテンプレート生成

目的を入力すると、リリィが小さなクエスト群を提案する。

- 例:
  - `睡眠を整えたい`
  - `Reactを学びたい`
  - `部屋を片付けたい`
  - `運動を再開したい`
- 出力例:
  - 5分でできるクエスト
  - 週1回のクエスト
  - 記録用クエスト
  - ふりかえり用クエスト
- 既存仕様との接続:
  - `create_quest`
  - スキル分類
  - クエスト作成画面
- 注意点:
  - いきなり大量作成せず、提案から選択して作成する

### 10. 集中モード連携

ユーザーが作業を始めたら、リリィが邪魔しないモードに入り、終わったあとに短く振り返る。

- 例:
  - 作業中は定期雑談を抑制する
  - 可聴タブや会議中は声かけを止める
  - 作業終了後に `今の集中、いい流れでした` と短くまとめる
- 入力候補:
  - ActivitySession
  - Chrome 可聴タブ snapshot
  - アイドル状態
  - カレンダー予定
- 期待効果:
  - 常駐キャラクターでありつつ、集中を妨げにくい
  - 行動ログと雑談のバランスが取りやすい

## 優先度の高そうな候補

### A. 今日の作戦会議

既存のホーム、クエスト、行動ログ、健康データを活かしやすく、リリィの価値が毎日の入口で伝わりやすい。

### B. クエスト自動棚卸し

TODO アプリとして重くなりがちなクエスト一覧を、リリィらしく軽く保てる。既存データだけでも始めやすい。

### C. 未完了の流れを拾う機能

行動ログ基盤の強みを活かしやすい。単なる記録閲覧ではなく、次の行動へ自然につながる。

## 今後仕様化するときの観点

- どのデータを正本にするか
- PWA と Desktop のどちらを主導にするか
- 自動実行か、ユーザー確認つき提案か
- 通知頻度と割り込み制御
- プライバシー設定、非表示、削除との整合
- JST 基準の日付・週・月の扱い
- AI 失敗時のフォールバック
- TDD で先に追加する受け入れテスト
