# 画面遷移スクロール仕様 2026-04-12

## 1. 目的

- Web アプリで画面遷移後に前画面のスクロール位置が残る不具合を防ぐ。
- タブ切り替え時や画面切り替え時に、常に先頭から内容を確認できるようにする。

## 2. 対象

- 下タブによる画面切り替え
- 画面遷移を伴う通常のルート変更
- URL クエリ変更で表現される画面内タブ切り替え

## 3. 仕様

- `pathname` または `search` が変化したタイミングで、スクロール位置をページ先頭へ戻す。
- スクロール位置のリセット対象は `window`、`document.documentElement`、`document.body` とする。
- クリア演出画面など個別画面で追加のスクロール制御が必要な場合でも、本仕様の共通リセットを前提としてよい。

## 4. テスト観点

- 別画面へ遷移したときにページ先頭へ戻ること
- 同一画面内でクエリ付きタブを切り替えたときにもページ先頭へ戻ること
