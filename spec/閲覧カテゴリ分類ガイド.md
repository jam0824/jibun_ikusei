# 閲覧カテゴリ分類ガイド

実際の閲覧履歴から抽出した、各カテゴリの具体例と判定基準。

---

## カテゴリ一覧と十分性の評価

現行の8カテゴリ（学習, 仕事, 健康, 生活, 創作, 対人, 娯楽, その他）で **十分カバーできる**。

ただし「その他」に流れやすい原因は、カテゴリの境界が不明確で AIが迷った際のデフォルトになっていること。以下のガイドで判定精度を改善する。

---

## 1. 学習

**定義**: 知識・スキルの習得を目的とした閲覧。仕事に直接関係しないもの。

**例**:
- 技術記事の閲読（Qiita, Zenn, はてなブログの技術記事）
- 技術書・学習プラットフォーム（O'Reilly, Udemy）
- カンファレンス資料の閲覧（Speaker Deck, Slideshare, Slido）
- YouTube の教育系動画（本要約, 科学解説, 語学学習, プログラミングチュートリアル）
- Wikipedia での調べもの
- 公式ドキュメントの閲読（AWS Docs, MDN, Google Cloud Docs）
- note.com の知識系・分析系記事
- ソフトウェアテストや品質保証の学習記事

**判定のコツ**:
- YouTube でもタイトルに「解説」「学習」「チュートリアル」「要約」「〜とは」があれば学習
- SNS 上でも、技術記事やカンファレンス情報へのリンクを含む投稿は学習

---

## 2. 仕事

**定義**: 業務遂行に直結する閲覧。開発、運用、プロジェクト管理など。

**例**:
- GitHub: PR, コミット, ワークフロー, Issues, ブランチ管理
- AWS Console: Lambda, DynamoDB, API Gateway, Cognito, IAM, CloudFormation
- Google Cloud Console: API管理, IAM, サービスアカウント
- AI アシスタント利用（Claude, ChatGPT）※開発目的の場合
- localhost での開発中アプリの確認
- Confluence, Jira などの業務ツール
- CI/CD のワークフロー確認
- 自社・自プロジェクトのアプリ動作確認（jam0824.github.io 等）

**判定のコツ**:
- クラウドコンソール（AWS, GCP, Azure）は基本的に仕事
- GitHub で自分のリポジトリを操作しているのも仕事
- AI ツール（Claude, ChatGPT）は利用目的で判断。開発関連なら仕事、調べものなら学習

---

## 3. 健康

**定義**: 心身の健康維持・改善に関する閲覧。

**例**:
- メンタルヘルスの記事（うつ病, ストレス, 気力回復）
- 運動・フィットネスの情報
- 栄養・食事管理サービス（nosh, 栄養モニタリング）
- 健康に関する YouTube 動画（脳科学, 休息法, 運動科学）
- 医療・健康系書籍の購入

**判定のコツ**:
- 「脳科学」「メンタル」「運動」「健康」「栄養」「睡眠」「回復」がタイトルに含まれれば健康
- 自己啓発系でも健康寄りのテーマなら健康

---

## 4. 生活

**定義**: 日常生活の維持・管理に関する閲覧。買い物、移動、家計など。

**例**:
- Amazon での日用品購入（コーヒー, 工具, 電池, 日用品）
- 楽天での買い物・ポイント管理
- 乗換案内（NAVITIME, Google Maps）
- 家計管理（Money Forward）
- メール確認（Gmail の通知処理・メルマガ）
- 保険・金融サービス
- スケジュール調整（調整さん）
- 福利厚生サービス（カフェテリアHQ）
- Google Workspace 管理

**判定のコツ**:
- Amazon は商品内容で判断。日用品・食品なら生活、漫画・ゲームなら娯楽、技術書なら学習
- メール一覧やログインページなどの汎用ページも、前後の文脈がなければ生活

---

## 5. 創作

**定義**: 何かを作り出す活動、または創作物の鑑賞に関する閲覧。

**例**:
- 個人サイト・ブログの管理（sakura.ne.jp, はてなブログ）
- アクセス解析（web拍手, Google Analytics）
- 画像編集・イラスト関連（dpi/mm変換, PSD/PNG ファイル操作）
- ドット絵・ゲームアセット制作
- 記事・コラムの執筆（Google ドキュメント）
- 掲示板・コミュニティサイトの運営管理
- 映画鑑賞（Amazon Prime Video, Netflix の映画作品）

**判定のコツ**:
- 「制作」「描く」「書く」「作る」「編集」に関わるツール利用は創作
- Google ドキュメントでの執筆は、テーマが技術なら学習寄りだが、「書くこと自体」が目的なら創作
- 映画は創作物の鑑賞として創作に分類する（アニメは娯楽）

---

## 6. 対人

**定義**: 人との交流・コミュニケーションが主目的の閲覧。

**例**:
- Facebook でのタイムライン閲覧・投稿・友達リクエスト
- mixi2 でのタイムライン閲覧・投稿
- Discord でのコミュニティ参加
- カンファレンスの参加・交流（JaSST, connpass）
- スケジュール調整（調整さん）でのご飯会・イベント調整

**判定のコツ**:
- Facebook, mixi2 は常に対人
- connpass, Discord はイベント参加目的なら対人
- X (Twitter) は対人ではなく **娯楽** に分類する（下記参照）

---

## 7. 娯楽

**定義**: 気晴らし・楽しみを主目的とした閲覧。

**例**:
- YouTube: ゆっくり解説（オカルト, 都市伝説, 歴史ミステリー）, ペット動画, ゲーム動画, お笑い・ネタ動画
- 動画配信サービス（Amazon Prime Video, Netflix）のアニメ視聴
- 漫画・ラノベの購入・閲覧（Kindle, Amazon の漫画セール）
- ゲーム関連（Steam ウィッシュリスト, ゲーム予約, ゲーム攻略）
- ホビー・フィギュア・コレクション（聖闘士星矢, プラモデル, キン肉マン消しゴム）
- TikTok の動画閲覧
- X (Twitter) のタイムライン閲覧・投稿・通知確認
- アダルトコンテンツ
- ニュースサイトの芸能・ゴシップ記事

**判定のコツ**:
- **X (Twitter) は常に娯楽** — タイムライン閲覧、通知確認、個別ポスト閲覧すべて含む
- YouTube の「ゆっくり解説」はテーマで判断。科学・技術テーマなら学習、オカルト・ミステリー・雑学なら娯楽
- Amazon Prime Video, Netflix などの動画視聴サービスは基本的に娯楽
- 漫画・ゲーム関連は基本的に娯楽
- Kindle セール巡回も娯楽

---

## 8. その他

**定義**: 上記のどれにも明確に当てはまらない場合のみ使用する。

**該当する例**:
- ブラウザ拡張の更新ページ（Adblock Plus）
- 汎用的なログイン・認証ページ（内容の手がかりがない場合）
- 検索エンジンのトップページ（「Google」のみ）
- 空白ページや意味のないURL

**「その他」にしてはいけない例**:
- タイトルから内容が推測できるページ → 内容に応じたカテゴリを選ぶ
- SNS のタイムライン → 対人
- Amazon のトップページ → 生活（買い物目的と推定）
- YouTube のトップページ → 娯楽（動画視聴目的と推定）

---

## 混在ドメインの判定ガイド

同一ドメインでもページ内容によってカテゴリが変わるドメインの判定指針。

### youtube.com
| タイトルのパターン | カテゴリ |
|---|---|
| 「解説」「チュートリアル」「学習」「要約」+ 技術/科学テーマ | 学習 |
| 「本要約」「〜の使い方」「NotebookLM」 | 学習 |
| 「脳科学」「休息」「運動」「健康」テーマ | 健康 |
| 「モチベーション」「習慣」「自己啓発」テーマ | 学習 |
| 「ゆっくり解説」+ オカルト/都市伝説/ミステリー | 娯楽 |
| ペット動画、ゲーム実況、お笑い | 娯楽 |
| トップページ（「YouTube」のみ） | 娯楽 |

### amazon.co.jp
| タイトルのパターン | カテゴリ |
|---|---|
| 日用品・食品（コーヒー, 工具, 電池） | 生活 |
| 技術書・ビジネス書の購入 | 学習 |
| 漫画・ラノベ・Kindleセール | 娯楽 |
| ゲーム・フィギュア・ホビー | 娯楽 |
| Prime Video / Netflix の映画 | 創作（創作物の鑑賞） |
| Prime Video / Netflix のドキュメンタリー | 学習 |
| Prime Video / Netflix のアニメ | 娯楽 |
| トップページ・カート・注文確認 | 生活 |

### x.com (Twitter/X)

**x.com は常に「娯楽」に分類する。** タイトルやポスト内容に関わらず、すべて娯楽とする。

### facebook.com / mixi.social (mixi2)

**facebook.com, mixi.social は常に「対人」に分類する。**

### note.com
| タイトルのパターン | カテゴリ |
|---|---|
| 技術・テスト・開発テーマ | 学習 |
| メンタルヘルス・健康テーマ | 健康 |
| エッセイ・コラム（人間関係テーマ） | 学習 |

### github.com
| タイトルのパターン | カテゴリ |
|---|---|
| 自分のリポジトリ操作（PR, commit, workflow） | 仕事 |
| 他者のリポジトリ閲覧・OSS 調査 | 学習 |
| GitHub トップページ | 仕事 |

---

## 判定の優先ルール

1. **迷ったら「その他」にしない** — 少しでもヒントがあれば最も近いカテゴリを選ぶ
2. **タイトルの内容 > ドメイン** — ドメインではなくタイトルの内容で判断する
3. **成長カテゴリ寄りに判断** — 学習と娯楽で迷ったら学習、仕事とその他で迷ったら仕事
4. **「その他」は最終手段** — タイトルからもドメインからも何も推測できない場合のみ
