# 食事管理アプリ 仕様整理 v3

## 1. 目的
個人利用の食事管理アプリを作成する。
バーコード検索・商品検索・料理検索・画像認識を使って食事候補を追加し、最終的に食事記録として保存する。

---

## 2. 検索機能

### 2.1 バーコード読み込み
- バーコード読み込みで対応商品を検索する
- データソースは OpenFoodFacts
- 検索結果は商品リストとして表示する

### 2.2 商品検索
- 商品検索欄で部分一致検索を行う
- データソースは OpenFoodFacts
- 検索結果は商品リストとして表示する
- 表示件数は最大100件

### 2.3 料理名検索
- 料理名を日本語入力で検索する
- 検索前に gpt-5-nano で英語翻訳する
- 検索先は Edamam
- 部分一致で料理候補を検索し、リスト表示する
- リスト表示名は 英語名 + 日本語翻訳
- 表示時の翻訳にも gpt-5-nano を使用する
- 表示件数は最大10件
- 件数制限の理由は翻訳コスト抑制のため

### 2.4 画像認識検索
- 画像認識で料理候補を抽出する
- データソースは Edamam
- 検索結果は料理リストとして表示する
- リスト表示名は 英語名 + 日本語翻訳
- 表示時の翻訳にも gpt-5-nano を使用する
- 表示件数は最大10件
- 件数制限の理由は翻訳コスト抑制のため
- 検索結果は複数選択可能

---

## 3. 登録画面
- 選択した商品・料理は料理登録画面に追加されていく
- 登録画面では数量・単位を編集できる
  - 商品: 1個、2個、100g など
  - 料理: 1皿、0.5皿、1.5人前 など
- 同一候補を複数追加できる前提で設計する

---

## 4. 保存単位
- 1回の登録は以下の食事区分のいずれかに紐づけて保存する
  - 朝食
  - 昼食
  - 夕食
  - 間食

---

## 5. 保存内容
- 「登録」ボタン押下で DB に保存する
- 保存対象:
  - 食事区分
  - 商品名 / 料理名
  - 数量 / 単位
  - 各アイテムの栄養素
  - 食事全体の栄養素合計値
- 合計値だけでなく、明細も保存する

---

## 6. 栄養素仕様

### 6.1 MVPで保存・表示する栄養素
- カロリー
- タンパク質
- 脂質
- 糖質
- カリウム
- カルシウム
- 鉄
- 食物繊維
- 飽和脂肪酸
- 塩分

### 6.2 MVPでは扱わない栄養素
- ビタミンA
- ビタミンE
- ビタミンB1
- ビタミンB2
- ビタミンB6
- ビタミンC

### 6.3 APIフィールド対応表
| 表示名 | 単位 | OpenFoodFacts | Edamam | 備考 |
|---|---:|---|---|---|
| カロリー | kcal | `energy-kcal` | `ENERC_KCAL` | そのまま採用 |
| タンパク質 | g | `proteins` | `PROCNT` | そのまま採用 |
| 脂質 | g | `fat` | `FAT` | そのまま採用 |
| 糖質 | g | `carbohydrates` を優先。無い場合は `carbohydrates-total - fiber` を使用 | `CHOCDF.net` | UI表示名は常に「糖質」 |
| カリウム | mg | `potassium` | `K` | そのまま採用 |
| カルシウム | mg | `calcium` | `CA` | そのまま採用 |
| 鉄 | mg | `iron` | `FE` | そのまま採用 |
| 食物繊維 | g | `fiber` | `FIBTG` | そのまま採用 |
| 飽和脂肪酸 | g | `saturated-fat` | `FASAT` | そのまま採用 |
| 塩分 | g | `salt` を優先。無い場合は `sodium × 2.54` | `NA` | Edamam の `NA` はナトリウムなので換算が必要 |

### 6.4 糖質の計算ルール
- アプリ上の表示名は「炭水化物」ではなく「糖質」とする
- Edamam は `CHOCDF.net` を糖質として採用する
- OpenFoodFacts は `carbohydrates` を糖質として優先採用する
- OpenFoodFacts で `carbohydrates` が無い場合は、`carbohydrates-total - fiber` を糖質として計算する
- `carbohydrates-total - fiber` の結果が 0 未満になる場合は 0 とする

### 6.5 塩分の計算ルール
- アプリ上の表示名は「塩分」とする
- OpenFoodFacts は `salt` をそのまま採用する
- OpenFoodFacts に `salt` が無く `sodium` がある場合は、`sodium × 2.54` で塩分を算出する
- Edamam は `NA` を使用し、以下で塩分に換算する
  - `塩分(g) = ナトリウム(g) × 2.54`
  - `塩分(g) = ナトリウム(mg) / 1000 × 2.54`

---

## 7. 未取得栄養素の扱い
- API が対象栄養素を返さなかった場合でも、その時点で取得できた値だけで合計値を計算する
- ただし、合計値が完全でない可能性があることを示すため、画面上に **「未取得あり」** と表示する
- このフラグは少なくとも以下の単位で保持できるようにする
  - 明細ごと
  - 食事ごとの合計
  - 1日ごとの合計

---

## 8. 登録完了画面
- その時に登録した栄養素の合計値を表示する
- 未取得栄養素を含む場合は「未取得あり」を表示する

---

## 9. 記録画面
- その日に食べた食事の栄養素合計値を表示する
- 栄養素はグラフで可視化する
- 一般的な40代男性の基準値と比較できるようにする
- 比較基準は日本の食事摂取基準ベースで設計する
- 40代男性は食事摂取基準の年齢区分上、**30〜49歳 男性** を使用する

### 9.1 画面構成
1. 日付ヘッダー
   - 表示対象日を表示する
   - 前日 / 翌日へ移動できる
2. 1日サマリー
   - その日の総摂取カロリーを表示する
   - 朝食 / 昼食 / 夕食 / 間食ごとの登録件数と小計カロリーを表示する
   - 未取得栄養素がある場合は「未取得あり」を表示する
3. 栄養素比較一覧
   - 10栄養素を一覧で表示する
   - 各行に以下を表示する
     - 栄養素名
     - 当日の合計値
     - 比較基準値
     - 達成状況
     - 比較グラフ
4. 栄養素詳細グラフ
   - 栄養素比較一覧で選択した1栄養素を大きく表示する
   - 初期選択はカロリーとする
5. 食事別内訳
   - 選択中の栄養素について、朝食 / 昼食 / 夕食 / 間食ごとの内訳を表示する

### 9.2 栄養素比較一覧の表示仕様
- 一覧の表示順は以下で固定する
  1. カロリー
  2. タンパク質
  3. 脂質
  4. 糖質
  5. 食物繊維
  6. 飽和脂肪酸
  7. 塩分
  8. カリウム
  9. カルシウム
  10. 鉄
- 1栄養素につき1行で表示する
- グラフは **横棒グラフ** を採用する
- 横棒グラフ上に、基準値の種類に応じた線または範囲帯を表示する
- 各行のステータスは以下のいずれかで表示する
  - 不足
  - 適正
  - 超過
  - 参考
  - 未取得あり

### 9.3 栄養素詳細グラフの仕様
- 詳細グラフは、選択した1栄養素のみを大きく表示する
- グラフ形式は **横棒グラフ + 基準線 / 基準帯** とする
- 実測値は1本の棒で表示する
- 朝食 / 昼食 / 夕食 / 間食の内訳は、同じ棒の中を食事区分ごとに分割して表示する
- 基準が単一値の場合は基準線を1本表示する
- 基準が範囲の場合は基準帯を表示する
- 上限のみの栄養素は、上限線を超えた分を超過として表示する
- 下限のみの栄養素は、下限線に対する達成率を表示する
- 未取得栄養素を含む場合は、グラフ付近に「未取得あり」を表示する

### 9.4 食事別内訳の表示仕様
- 選択中の栄養素について、以下の表を表示する
  - 朝食: 値
  - 昼食: 値
  - 夕食: 値
  - 間食: 値
  - 合計: 値
- 該当する食事記録がない区分は 0 と表示する
- 未取得栄養素がある区分は「未取得あり」を併記する

### 9.5 比較基準値（30〜49歳 男性）
- エネルギー
  - 基準の種類: 推定エネルギー必要量
  - 基本表示値: **2,750 kcal/日**
  - 備考: 身体活動レベル「ふつう」を採用
  - 参考: 低い 2,350 / ふつう 2,750 / 高い 3,150 kcal/日
- タンパク質
  - 基準の種類: 推奨量
  - **65 g/日**
  - 評価方法: **65 g以上で適正**
- 脂質
  - 基準の種類: 目標量
  - **20〜30 %エネルギー**
  - グラフ比較用の換算値（2,750 kcal/日基準）: **61〜92 g/日**
  - 評価方法: **61〜92 gで適正**
- 糖質
  - 比較表示名: **糖質（炭水化物相当参考値）**
  - 公式な糖質基準は使わず、炭水化物基準を参考表示する
  - 炭水化物の基準: **50〜65 %エネルギー**
  - 2,750 kcal/日換算: **344〜447 g/日**
  - 評価方法: **参考値として表示し、厳密な適正判定には使わない**
- カリウム
  - 基準の種類: 目標量
  - **3,000 mg/日以上**
  - 評価方法: **3,000 mg以上で適正**
- カルシウム
  - 基準の種類: 推奨量
  - **750 mg/日**
  - 評価方法: **750 mg以上で適正**
- 鉄
  - 基準の種類: 推奨量
  - **7.5 mg/日**
  - 評価方法: **7.5 mg以上で適正**
- 食物繊維
  - 基準の種類: 目標量
  - **22 g/日以上**
  - 評価方法: **22 g以上で適正**
- 飽和脂肪酸
  - 基準の種類: 目標量
  - **7 %エネルギー以下**
  - グラフ比較用の換算値（2,750 kcal/日基準）: **21 g/日以下**
  - 評価方法: **21 g以下で適正**
- 塩分
  - 基準の種類: 目標量
  - **7.5 g/日未満**
  - 評価方法: **7.5 g未満で適正**
  - 補足: 高血圧・CKD の重症化予防では **6.0 g/日未満** を参考表示可能

### 9.6 ステータス判定ルール
- 参考
  - カロリー
  - 糖質（炭水化物相当参考値）
- 不足
  - 下限 / 推奨量に達していない場合
- 適正
  - 下限以上、上限以下、または範囲内の場合
- 超過
  - 上限を超えた場合、または範囲上限を超えた場合
- 未取得あり
  - 取得欠損を含む場合に追加表示する

### 9.7 グラフのスケール仕様
- 横棒グラフの最大値は、以下の優先順位で決める
  1. 実測値
  2. 基準上限値または基準値
  3. 上記の大きい方の **120%**
- 上限のみの栄養素では、上限値の120%を基本スケールとする
- 範囲表示の栄養素では、範囲上限の120%を基本スケールとする
- 実測値がスケールを超える場合は、その値に合わせて自動拡張する

### 9.8 記録画面のMVP方針
- MVPでは、1日単位の表示に限定する
- 週次 / 月次の推移グラフは扱わない
- 比較対象は 30〜49歳 男性で固定する
- 個人の年齢・活動量による自動補正は行わない

## 10. 画面遷移

### 10.1 画面一覧
1. 記録画面
2. 食事追加画面
3. バーコードスキャンモーダル
4. 画像入力モーダル
5. 登録画面
6. 登録完了画面

### 10.2 起動時の初期表示
- アプリ起動時は **記録画面** を初期表示とする
- 初期日付は「今日」とする

### 10.3 基本遷移
```text
記録画面
  └─ 食事区分の「追加」ボタン
       ↓
    食事追加画面
      ├─ バーコード読み込み → バーコードスキャンモーダル → 食事追加画面
      ├─ 商品検索 → 食事追加画面内で結果表示
      ├─ 料理名検索 → 食事追加画面内で結果表示
      └─ 画像認識 → 画像入力モーダル → 食事追加画面内で結果表示
       ↓
    登録画面
       ↓
    登録完了画面
       ↓
    記録画面
```

### 10.4 記録画面
- 役割
  - その日の食事記録を確認するホーム画面
  - 栄養素比較グラフを確認する起点画面
  - 新しい食事を追加する起点画面
- 主な表示要素
  - 日付ヘッダー
  - 1日サマリー
  - 栄養素比較一覧
  - 栄養素詳細グラフ
  - 食事別内訳
  - 朝食 / 昼食 / 夕食 / 間食 のセクション
- 遷移
  - 日付の前日 / 翌日操作で同画面内の日付切り替えを行う
  - 朝食 / 昼食 / 夕食 / 間食 の各セクションにある **「追加」** ボタン押下で、該当食事区分を指定した状態で **食事追加画面** に遷移する
- MVP方針
  - 記録済みデータの編集・削除はこの時点では扱わない
  - 記録済み明細の詳細画面は作らず、記録画面内で一覧表示する

### 10.5 食事追加画面
- 役割
  - 食事候補を検索し、登録対象を一時的に集める画面
- 画面上部
  - 対象日付
  - 対象食事区分（朝食 / 昼食 / 夕食 / 間食）
- 食事区分
  - 記録画面から渡された食事区分を初期値として表示する
  - この画面では変更しない
  - 食事区分を変えたい場合は、戻って別の「追加」ボタンから入り直す
- 検索タブ
  - バーコード
  - 商品検索
  - 料理名検索
  - 画像認識
- 遷移と挙動
  - タブ切り替えは同一画面内で行う
  - タブを切り替えても、その登録セッション中に追加した候補は保持する
  - 1件以上追加されている場合のみ **「登録画面へ」** ボタンを活性化する
  - 戻る操作時に未登録の候補がある場合は、破棄確認ダイアログを表示する

#### 10.5.1 バーコードタブ
- **「スキャンする」** ボタン押下で **バーコードスキャンモーダル** を開く
- スキャン成功後は OpenFoodFacts で検索し、結果を食事追加画面内に表示する
- 検索結果から候補を追加できる

#### 10.5.2 商品検索タブ
- キーワード入力で OpenFoodFacts を検索する
- 部分一致で候補を表示する
- 結果は最大100件まで表示する
- 各候補には **「追加」** ボタンを表示する
- 追加した候補は登録セッションに保持する

#### 10.5.3 料理名検索タブ
- 日本語入力を受け付ける
- 検索前に gpt-5-nano で英語翻訳する
- Edamam で検索する
- 結果は最大10件まで表示する
- 各候補は **英語名 + 日本語翻訳** で表示する
- 各候補には **「追加」** ボタンを表示する
- 追加した候補は登録セッションに保持する

#### 10.5.4 画像認識タブ
- **「写真を選ぶ」** または **「撮影する」** 操作で **画像入力モーダル** を開く
- 画像選択後に Edamam で画像認識を行い、候補を食事追加画面内に表示する
- 結果は最大10件まで表示する
- 各候補は **英語名 + 日本語翻訳** で表示する
- 画像認識結果は複数選択可能とする
- **「選択した候補を追加」** で登録セッションに保持する

### 10.6 バーコードスキャンモーダル
- 役割
  - カメラでバーコードを読み取る
- 遷移
  - 成功時は前画面である **食事追加画面** に戻る
  - キャンセル時も **食事追加画面** に戻る
- MVP方針
  - 1回のスキャンで1つのコードを読み取る
  - 連続スキャンは扱わない

### 10.7 画像入力モーダル
- 役割
  - カメラ撮影またはフォトライブラリ選択で画像を取得する
- 遷移
  - 画像選択後は前画面である **食事追加画面** に戻り、認識結果を表示する
  - キャンセル時は **食事追加画面** に戻る
- MVP方針
  - 1回の認識で1枚の画像のみ扱う
  - 連続画像投入は扱わない

### 10.8 登録画面
- 役割
  - 追加した候補を確認し、数量・単位を編集して登録する
- 主な表示要素
  - 対象日付
  - 対象食事区分
  - 選択済み候補の一覧
  - 各候補の数量・単位編集UI
  - 各候補の削除ボタン
  - 食事全体の栄養素合計
  - 未取得あり表示
- 遷移
  - **「追加画面に戻る」** で **食事追加画面** に戻る
  - 戻る場合も登録セッションは保持する
  - **「登録」** ボタン押下で保存処理を行い、成功後に **登録完了画面** に遷移する
- MVP方針
  - 対象食事区分は表示のみとし、この画面では変更しない
  - 数量・単位の編集と削除のみを扱う

### 10.9 登録完了画面
- 役割
  - 今回保存した内容を確認する
- 主な表示要素
  - 対象日付
  - 対象食事区分
  - 登録した明細一覧
  - 今回登録分の栄養素合計値
  - 未取得あり表示
- 遷移
  - **「記録画面へ戻る」** で **記録画面** に戻る
  - 戻った際は対象日付を維持し、該当食事区分までスクロールした状態で表示する
- セッション処理
  - 登録完了時点で、一時的に保持していた登録セッションは破棄する

### 10.10 画面遷移ルール
- 1回の登録セッションでは、複数の検索方法を横断して候補を追加できる
  - 例: 商品検索で2件追加 + 料理名検索で1件追加 + 画像認識で2件追加
- 追加候補は、登録完了または破棄するまで保持する
- 登録前に戻る場合は、未保存データ破棄の確認を出す
- 登録完了後は、前回の検索条件・候補・画像は保持しない

### 10.11 MVPで扱わない遷移
- 記録済みデータの編集画面
- 記録済みデータの削除確認画面
- 週次 / 月次の推移画面
- ユーザー設定画面
- 比較対象（年齢・性別・活動量）の切り替え画面

## 11. この時点で確定した主要仕様
1. 商品検索は OpenFoodFacts を使う
2. 料理名検索・画像認識は Edamam を使う
3. 料理名検索入力は日本語、検索時は gpt-5-nano で英語翻訳する
4. 料理名・画像認識の表示名は 英語名 + 日本語翻訳 とする
5. 料理名検索と画像認識の結果件数は最大10件
6. 商品検索結果の件数は最大100件
7. 登録画面では数量・単位の編集を可能にする
8. 保存単位は 朝食 / 昼食 / 夕食 / 間食
9. DB には合計値だけでなく明細も保存する
10. MVP の栄養素は10項目に限定する
11. 糖質と塩分は API差分を吸収する変換ルールを持つ
12. 栄養素が欠損していても取得できた値で合計し、「未取得あり」を表示する
13. 比較基準は 30〜49歳 男性の日本人の食事摂取基準を使う
14. 糖質は「炭水化物相当の参考値」と明示して比較表示する
15. 記録画面を起点に、食事追加 → 登録 → 登録完了 の流れで遷移する
