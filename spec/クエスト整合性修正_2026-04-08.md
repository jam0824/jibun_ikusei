# クエスト整合性修正 追補仕様

更新日: 2026-04-08 (JST)

## 背景

- 記録画面で、完了記録の `questId` に対応する quest を引けない場合は `削除されたクエスト` を表示する。
- API の `PUT /quests/{id}` が部分更新ではなく実質的な全置換になっていると、`status` だけ更新したつもりでも `title` などが失われうる。
- API の `POST /completions` が参照先 quest の存在を検証しないと、削除済みまたは不整合な `questId` を持つ completion が作られうる。

## 仕様

### 1. `PUT /quests/{id}`

- 既存の quest を読み込んでから更新内容をマージする。
- リクエストに含まれない既存フィールドは保持する。
- `updatedAt` は更新時刻で上書きする。
- `createdAt` は既存値を優先して保持する。

### 2. `POST /completions`

- `questId` に対応する quest が存在することを必須とする。
- 参照先 quest は `active` 状態でなければならない。
- quest が存在しない、`title` が欠損している、または `active` ではない場合は completion を作成せず `400` を返す。
- フロントエンドが完了記録を楽観追加していた場合でも、`400` を受けたら当該 completion・関連メッセージ・その完了で派生したローカル補助データを巻き戻す。
- 巻き戻し時は対象 quest をクラウドの最新一覧で再確認し、存在しなければローカルからも除外する。

### 3. 起動時の整合性回復

- クラウド同期時、クラウドに存在しないローカル専用 completion のうち、対応 quest をローカルでも引けない orphan completion は除去する。
- orphan completion に紐づく Lily メッセージも合わせて除去する。

## 期待される効果

- `status` のみ更新した quest で `title` が消えない。
- 削除済み・不整合な quest を参照する completion の新規作成を防ぐ。
- 記録画面で新たに `削除されたクエスト` が増殖する経路を遮断する。
- 端末ローカルにだけ残った不整合な完了記録が、次回同期で記録画面に残り続けない。
